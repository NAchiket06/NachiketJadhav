Dijkstra's Algorithm

Dijkstra's algorithm is a widely used pathfinding algorithm that finds the shortest path between two nodes in a graph by repeatedly selecting the node with the lowest cumulative cost from a source node.

It maintains a priority queue to systematically explore and update neighboring node costs, making it a foundational tool for network routing, transportation planning, and optimal path determination.

How Does It Work?

Graph setup, pathfinding, and curve smoothing for traversable routes.

01

Graph Setup

The algorithm needs a weighted graph where each node has a list of neighboring nodes. A selected node can move to its neighbor nodes after visiting it, and the setup can be extended to give each neighbor a weight property for prioritization.

02

Complete Graph

Once all nodes are selected, the environment becomes a complete graph that the algorithm can search through to determine valid movement options.

03

Pathfinding

After graph setup, the start and end nodes are marked, and the algorithm finds a path between them. The path visualization shows the route found between the source and target nodes.

Reference: Dijkstra's Algorithm

04

Bezier Curves

For smooth traversal, hard edges in the path are softened by calculating Bezier curve points between neighboring nodes. A step value controls how smooth the curve needs to be.

Reference: Bezier Curves Explanation

The Result

A smooth path between two nodes for entities to follow through virtual environments.

Pathfinding For Pedestrians

Applying the algorithm in a city environment for moving agents.

The algorithm is then used by cars in a city environment to move from one point to another while avoiding other cars, following traffic rules, and stopping at red signals.
